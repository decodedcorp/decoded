## 3. UI 요소 정의

| UI ID | 구분 | 요소명 | 속성/상태 | 인터랙션/로직 |
|:---:|:---:|:---|:---|:---|
| **ICON-BACK** | 아이콘 | 뒤로가기 | - Icon: Arrow Left<br>- Size: 24×24px | **Click**: `/profile`로 이동 |
| **TXT-AMOUNT** | 텍스트 | 출금 가능액 | - Font: H1 (32px Bold)<br>- Color: `text-primary`<br>- Format: Currency (₩) | 현재 출금 가능한 확정 금액 표시 |
| **TXT-PENDING** | 텍스트 | 대기 금액 | - Font: Body (14px)<br>- Color: `text-warning` | 아직 확정되지 않은 금액 (당월 클릭 수익) |
| **TXT-PROCESSING** | 텍스트 | 처리 중 금액 | - Font: Body (14px)<br>- Color: `text-muted` | 출금 요청 후 처리 중인 금액 |
| **INP-AMOUNT** | 입력 | 출금 금액 | - Type: Number<br>- Placeholder: "₩ 0"<br>- Format: Currency | **Input**: 유효성 검사<br>1. 잔액 초과 시 에러<br>2. 최소 금액 미만 시 에러 |
| **SEL-METHOD** | 셀렉트 | 출금 방식 | - Options: Bank Transfer, PayPal | **Change**: 하단 입력 폼 변경<br>- Bank: 은행 + 계좌번호<br>- PayPal: 이메일 주소 |
| **SEL-BANK** | 셀렉트 | 은행 선택 | - Options: 국내 주요 은행 목록 | Bank Transfer 선택 시만 표시 |
| **INP-ACCOUNT** | 입력 | 계좌번호 | - Type: Text<br>- Placeholder: "계좌번호 입력" | Bank Transfer 선택 시만 표시 |
| **BTN-WITHDRAW** | 버튼 | 출금 요청 | - Style: Primary<br>- Disabled: 유효값 미입력 시 | **Click**: 출금 API 호출<br>→ 성공: 확인 모달 + 잔액 차감<br>→ 실패: 에러 메시지 표시 |
| **SEC-HISTORY** | 섹션 | 출금 내역 | - Title: "📋 Withdrawal History" | - |
| **LIST-ITEM** | 리스트 | 출금 내역 항목 | - Amount + Status + Date<br>- Status Color 구분 | - |

### 상태별 색상 코드

| 상태 | 색상 | 의미 |
|:---|:---|:---|
| Pending | `text-warning` (Orange) | 출금 요청 접수됨 |
| Processing | `text-info` (Blue) | 처리 진행 중 |
| Completed | `text-success` (Green) | 완료됨 |
| Rejected | `text-danger` (Red) | 거절됨 |

## 4. 상태 정의

| 상태 | 조건 | UI 변화 |
|:---|:---|:---|
| **기본** | 데이터 로딩 완료 | 잔액 및 폼 정상 표시 |

| **에러** | API 실패 | 에러 토스트 표시 |

## 5. 데이터 요구사항

### 5.1 API 호출

| API | Method | Endpoint | 호출 시점 | 응답 |
|:---|:---:|:---|:---|:---|
| 수익 조회 | GET | `/api/profile/earnings` | 페이지 진입 시 | `{ available, pending, processing }` |
| 출금 내역 | GET | `/api/profile/withdrawals` | 페이지 진입 시 | `{ withdrawals: Withdrawal[] }` |
| 출금 요청 | POST | `/api/profile/earnings/withdraw` | 버튼 클릭 시 | `{ success, withdrawalId }` |

### 5.2 상태 관리

| 스토어 | 키 | 타입 | 설명 |
|:---|:---|:---|:---|
| React Query | `["profile", "earnings"]` | `QueryKey` | 수익 데이터 캐시 |
| React Query | `["profile", "withdrawals"]` | `QueryKey` | 출금 내역 캐시 |
| Local State | `formData` | `WithdrawalForm` | 폼 입력 상태 |

### 5.3 데이터 타입

```typescript
interface EarningsData {
  available: number;   // 출금 가능 금액
  pending: number;     // 대기 중 금액
  processing: number;  // 처리 중 금액
}

interface Withdrawal {
  id: string;
  amount: number;
  status: 'pending' | 'processing' | 'completed' | 'rejected';
  paymentMethod: 'bank_transfer' | 'paypal';
  createdAt: Date;
  completedAt?: Date;
  rejectedReason?: string;
}

interface WithdrawalForm {
  amount: number;
  paymentMethod: 'bank_transfer' | 'paypal';
  bank?: string;
  accountNumber?: string;
  paypalEmail?: string;
}
```

## 6. 에러 처리

| 에러 코드 | 상황 | 사용자 메시지 | 처리 방법 |
|:---:|:---|:---|:---|
| 400 | 잘못된 요청 | "입력 정보를 확인해주세요" | 필드 에러 표시 |
| 401 | 인증 만료 | "세션이 만료되었습니다" | 로그인 페이지로 이동 |
| 403 | 출금 권한 없음 | "출금 권한이 없습니다" | 고객센터 안내 |
| 422 | 잔액 부족 | "출금 가능 금액이 부족합니다" | 잔액 새로고침 |
| 429 | 요청 제한 | "잠시 후 다시 시도해주세요" | 5초 후 재시도 가능 |
| 500 | 서버 오류 | "일시적인 오류가 발생했습니다" | 재시도 버튼 표시 |

## 7. 접근성 (A11y)

- **키보드 네비게이션**: Tab으로 모든 폼 필드 접근
