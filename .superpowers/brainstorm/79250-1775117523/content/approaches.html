A) Smart Ratio Bucketing (Instagram-inspired)

Images classified into 3 buckets: portrait (3:4), square (1:1), landscape (4:3). Grid stays uniform per row. Minimal crop, clean alignment.

Pros

  • Cleanest grid alignment
  • Simplest implementation
  • Fast rollback (just change bucket logic)

Cons

  • Still some crop on odd ratios
  • Less visual variety

B) True Masonry (Pinterest-inspired)

Each card height = actual image ratio. Zero crop. Visual variety from natural ratio differences. Cards fill columns organically.

Pros

  • Zero crop, every image fully shown
  • Most visual variety
  • Already have MasonryGrid component

Cons

  • Layout shift until dimensions known
  • Ragged edges between columns
  • Extreme ratios need clamping

FEED: Bucket
Different strategy per context → best of both

C) Hybrid: Masonry + Bucketing (Recommended)

Explore/Grid pages use true masonry (zero crop). Feed/Profile use ratio bucketing (clean rows). Each context gets the best strategy.

Pros

  • Best fit per context
  • Explore: full image beauty
  • Feed: clean alignment
  • Feature flag per component

Cons

  • Two display modes to maintain
  • Slightly more complex
