## Milestone Summary

**Key Decisions:**
- Admin layout completely separate from main app (no shared header/footer/navbar)
- Defense-in-depth: middleware + layout server-side double-check for admin auth
- Deterministic djb2 hash-based mock data for stable dashboard values
- Real-data-first pattern: query Supabase first, fallback to mock
- Admin API response shape: bare entity objects (no `{ data: }` wrapper)
- Admin API auth: always enforced (no NODE_ENV conditional bypass)
- Recharts for all chart visualizations (AreaChart, BarChart)
- URL-synced pagination and filtering across all admin pages
- Custom streaming hook (not React Query) for realistic tail-f behavior

**Issues Resolved:**
- Pipeline accordion crash (PIPE-02): response shape was `{ data: pipeline }` but hook expected bare `PipelineExecution`
- Pipeline retry unreachable (PIPE-03): unblocked by PIPE-02 fix
- Auth inconsistency: removed `NODE_ENV !== "development"` bypass from `ai-cost/kpi` and `dashboard/stats` routes

**Issues Deferred:**
- None (all audit gaps closed in v3-06)

**Technical Debt Incurred:**
- 3x `as any` type assertions for Supabase table name mismatch (post vs posts)
- AUDIT-03 edits are client-side only — no PATCH API route (intentional per mock-data scope)
- Orphaned `scrollRef` export from `useLogStream` (unused by LogStream component)
- `useServerLogs` queryKey missing `to` param (latent stale-cache bug, currently inert)
- `PipelineTable` map returns React Fragment without key (console warning)
- Admin logout does not redirect (user stays on /admin UI after signOut)
