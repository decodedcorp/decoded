  <action>
    **1. MagazineRenderer.tsx** -- The core layout engine. "use client".

    Props:
    ```typescript
    interface MagazineRendererProps {
      issue: MagazineIssue;
      className?: string;
    }
    ```

    Implementation:
    - On mount, call `injectMagazineTheme(issue.theme_palette, containerRef.current)` to set CSS custom properties on the container div.
    - On unmount, call `removeMagazineTheme(containerRef.current)`.
    - Set container to `position: relative`, `min-height: 100vh`, `bg-mag-bg`, `text-mag-text`.
    - Iterate `issue.layout_json.components[]`:
      - For each component, call `getComponent(component.type)` from registry
      - If null (unknown type), skip silently
      - Render component wrapped in a positioned div:
        ```tsx
        <div
          ref={(el) => { componentRefs.current[i] = el; }}
          style={{
            position: 'absolute',
            left: `${component.x}%`,
            top: `${component.y}%`,
            width: `${component.w}%`,
            height: component.h ? `${component.h}%` : 'auto',
          }}
        >
          <Component data={component.data} />
        </div>
        ```
      - Note: For mobile, consider using `position: relative` with flow layout instead of absolute when viewport is 'mobile'. The absolute positioning from layout_json is the spec, but add a wrapper div that sets the container height based on the max(y + h) of all components.

    **GSAP Orchestration:**
    - Register ScrollTrigger plugin: `gsap.registerPlugin(ScrollTrigger)` at module top
    - Create GSAP context scoped to container ref
    - Build a master timeline with staggered entry based on each component's `animation_type`:
      - `'fade-up'`: `{ opacity: 0, y: 30 }` -> `{ opacity: 1, y: 0, duration: 0.6 }`
      - `'scale-in'`: `{ opacity: 0, scale: 1.1 }` -> `{ opacity: 1, scale: 1, duration: 1.8 }` (hero)
      - `'slide-left'`: `{ opacity: 0, x: 40 }` -> `{ opacity: 1, x: 0, duration: 0.5 }`
      - `'parallax'`: Set up ScrollTrigger with `{ y: -50 * (parallax_depth || 0.2) }` on scroll
      - `'none'`: Just set opacity to 1, no animation
    - Use `animation_delay` to offset timeline position
    - After entry animation completes, interactive states are enabled (items are visible and clickable)
    - On unmount: `gsapContext.revert()` in useEffect cleanup

    **Important GSAP notes:**
    - Import gsap and ScrollTrigger: `import gsap from 'gsap'` and `import { ScrollTrigger } from 'gsap/ScrollTrigger'`
    - Use `useLayoutEffect` (or `useEffect` with SSR guard) for GSAP setup
    - All animation targets use refs collected during render
