  <action>
    All "use client" components. Import gsap and register ScrollTrigger if needed.

    **1. DecodingParticles.tsx:**
    - Props: `{ isActive: boolean, imageUrls?: string[] }`
    - Renders 8-12 small circular elements (60x60px) representing user's reference images
    - Uses picsum.photos thumbnails as particle content (or solid accent-colored circles if no imageUrls)
    - When `isActive`:
      - Position each particle at random edge locations (top/bottom/left/right of viewport)
      - GSAP `.to()` each particle toward center of screen with staggered delays (0.2-0.5s interval)
      - Duration: 2-4s per particle, ease "power2.inOut"
      - On reaching center: scale down to 0 and fade out (particle "absorbed")
      - Loop: after all particles reach center, restart with new random positions
    - When `!isActive`: clean up, hide particles
    - GSAP context for cleanup on unmount
    - Use absolute positioning within a full-viewport container (`fixed inset-0`)

    **2. StyleKeywordChip.tsx:**
    - Props: `{ keyword: string, delay?: number }`
    - Small pill/chip with text: `#${keyword}`
    - Styled: `bg-transparent border border-mag-accent text-mag-accent px-3 py-1 rounded-full text-sm`
    - Entry animation: scale 0->1, opacity 0->1, 0.3s, with accent box-shadow glow (`shadow-[0_0_12px_var(--mag-accent)]`)
    - Apply `animation_delay` from props
    - GSAP `.from()` on mount with delay

    **3. DecodingText.tsx:**
    - Props: `{ phrases: string[], isActive: boolean }`
    - Cycles through phrases with fade transition every 2.5s
    - **Variable font weight animation:** Use GSAP to animate CSS `font-variation-settings` or `font-weight` from 300 to 900 in a 1.5s loop
    - If variable font not available, use GSAP to animate `font-weight` between 300 and 700 (stepped via `snap`)
    - Text color: `text-mag-text`
    - Font size: large (text-2xl md:text-3xl), centered
    - When `!isActive`: stop animation

    **4. ProgressGlow.tsx:**
    - Props: `{ progress: number }` (0-100)
    - Horizontal bar at bottom of ritual area
    - Bar fill: `bg-mag-accent` with `scaleX` transform based on progress/100
    - `transform-origin: left`
    - Percentage text below bar
    - Accent glow on the bar: `shadow-[0_0_15px_var(--mag-accent)]`
    - Width: 80% centered, height: 3px
    - No GSAP needed -- progress is driven by parent via props (GSAP in parent animates the progress value)

    **5. DecodingRitual.tsx:**
    - Props: `{ isActive: boolean, onComplete: () => void }`
    - Full-screen overlay (`fixed inset-0 z-50 bg-mag-bg`)
    - **Orchestration timeline when `isActive`:**
      1. Fade in overlay (opacity 0->1, 0.3s)
      2. Start `DecodingParticles` (isActive=true)
      3. After 1s: start showing `StyleKeywordChip` components (stagger 0.3s each)
         - Mock keywords: ["Minimal", "Streetcore", "Cyberpunk", "Monochrome", "Avant-garde"]
      4. Simultaneously: `DecodingText` cycles through ["Analyzing your DNA...", "Decoding your style...", "Curating your layout...", "Almost there..."]
      5. `ProgressGlow` driven by GSAP tween: animate a local `progress` ref from 0 to 100 over ~6 seconds
         - Front-loaded easing: use `ease: "power3.out"` so 0-80% is fast, 80-100% is slow
      6. When progress reaches 100: call `onComplete()` after 0.5s delay
    - When `isActive` becomes false: fade out overlay (0.3s)
    - GSAP context scoped to overlay container, reverted on unmount or deactivation
  </action>
