    - If `item.spots` exists and has items:
      - On hover, show spot markers at (x%, y%) positions over the image

      - On marker hover: show a small popup tooltip above with `label`, `brand`, `price`
      - Popup: `bg-mag-bg/90 backdrop-blur-sm border border-mag-accent/30 text-mag-text text-xs px-3 py-2 rounded-lg`
      - GSAP: markers fade in with stagger 0.05s on card hover
    - If no spots: just show the card normally, no overlay logic

    **Parallax (GSAP ScrollTrigger):**
    - Each item gets subtle parallax on scroll: `y: -20 * (0.5 + Math.random() * 0.5)` with scrub
    - ScrollTrigger: trigger is the item itself, start "top bottom", end "bottom top", scrub 1
    - Use `useEffect` with GSAP context, revert on unmount

    **Entry animation:**
    - Initial: `opacity: 0, y: 40`
    - ScrollTrigger driven: animate to `opacity: 1, y: 0` when entering viewport
    - Use stagger based on `index` prop: `delay: index * 0.08`

    Props: `items: GridItemData[]`, `className?: string`

    Implementation:
    - Use CSS `columns` for true masonry layout:
      ```
      columns-1 sm:columns-2 lg:columns-3 xl:columns-4
      gap-4
      ```
    - Each MasonryGridItem wrapped in `break-inside-avoid mb-4`
    - Section title: "DECODED PICKS" with accent underline, positioned above the grid
    - Use `bg-mag-bg` for dark background continuity
    - Pass `index` to each MasonryGridItem for stagger calculation

    Why CSS columns over CSS Grid: CSS `columns` gives true masonry behavior (items fill vertically, then wrap to next column) without JavaScript measurement. CSS Grid masonry is not widely supported. No library needed.
