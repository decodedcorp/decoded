    **1. `SmartNav.tsx`** -- Scroll-responsive navigation overlay. "use client".

    Props: `className?: string`

    Implementation:
    - Fixed position nav bar at top of viewport (`fixed top-0 left-0 right-0 z-50`)
    - Background: `bg-mag-bg/80 backdrop-blur-md` when visible
    - Content: "DECODED" logo text (left), minimal nav links (right: Magazine, Explore, Profile)
    - Logo text: `text-mag-accent font-bold tracking-[0.3em] text-sm`

    **Scroll behavior (GSAP ScrollTrigger):**
    - Track scroll direction using ScrollTrigger.observe or manual scroll tracking
    - On scroll DOWN past 100px: translate nav up (hide) with `gsap.to(navRef, { y: -100, duration: 0.3 })`
    - On scroll UP: translate nav back (show) with `gsap.to(navRef, { y: 0, duration: 0.3 })`
    - At very top (scrollY < 50): fully transparent background, always visible
    - Transition: smooth 0.3s

    **Implementation approach:**
    - Use `useEffect` with scroll event listener (throttled with requestAnimationFrame)
    - Track `lastScrollY` ref to determine direction
    - GSAP for smooth animation of the hide/show
    - Cleanup: remove scroll listener on unmount

    **Important:** This SmartNav overlays ON TOP of the existing ConditionalNav from layout.tsx. The page should signal to ConditionalNav to hide (or SmartNav should visually override it). Simplest approach: set `data-hide-nav` attribute on the page container, and if ConditionalNav doesn't support this, just let SmartNav sit on top with higher z-index. The existing nav uses z-index from the design system -- check and use z-50 or higher.
