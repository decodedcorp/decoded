## Duplicate Implementations

### Critical: LoginCard

**Design-system version:** `lib/design-system/login-card.tsx`
- Pure presentational component
- Props: `onOAuthLogin`, `onGuestLogin`, `loadingProvider`, `error`

**App version (ACTIVE):** `lib/components/auth/LoginCard.tsx`
- Connected component with `useAuthStore` hook
- Directly calls store methods

**Recommendation:** Keep design-system as presentational, pass handlers from page level.

### Critical: StepIndicator

**Design-system version:** `lib/design-system/step-indicator.tsx`
- 3 size variants, 3 visual variants, connector lines

**App version (ACTIVE):** `lib/components/request/StepIndicator.tsx`
- Hardcoded 3 steps, simple dots only

**Recommendation:** Migrate request flow to design-system version.

### Critical: Hotspot vs SpotMarker

**Design-system Hotspot:** `lib/design-system/hotspot.tsx`
- Generic position-based marker with variants

**App SpotMarker (ACTIVE):** `lib/components/request/SpotMarker.tsx`
- Tightly coupled to DetectedSpot type

**Recommendation:** Evaluate if SpotMarker can use Hotspot internally.

## Recommendations

### Option A: Complete Milestone (Accept Tech Debt)

Mark v2.1 as complete. Track orphaned components in backlog for future integration.

**Pros:**
- Design-system components exist and are ready
- Core milestone goal (component creation) achieved
- Can integrate incrementally over time

**Cons:**
- Duplicate implementations will diverge
- Orphaned components may become stale
- 35% integration rate is low

### Option B: Add Integration Phase (v2.1-07)

Create a consolidation phase to integrate orphaned components.

**Scope:**
1. Migrate login page to design-system components (OAuthButton, LoginCard, Divider)
2. Migrate request flow to design-system components (StepIndicator, Hotspot)
3. Integrate profile components (Badge, LeaderItem, RankingItem)
4. Create EmptyState/ErrorState design-system components
5. Remove duplicate implementations

**Expected Outcome:** Integration score 65% → 90%
