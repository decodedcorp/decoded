## Integration Gaps

### MISS-01: turbo.json test task missing codegen dependency

- **Severity:** Minor
- **Affected:** CI-01, TEST-01
- **Issue:** `turbo.json` has `"test": {}` with no `dependsOn`. In a clean CI `turbo run test`, the Zod validation test will fail because `lib/api/generated/zod/decodedApi.zod.ts` is gitignored and must be generated first.
- **Fix:** Add `"dependsOn": ["generate:api"]` to the `test` task in `turbo.json`

## Tech Debt

### TD-01: StyleCard.tsx solution_count field (Phase 41)
`(s as unknown as { solution_count?: number }).solution_count ?? 0` — backend OpenAPI spec does not include `solution_count` in `SpotListItem`. Type cast workaround applied. Silently renders `hasSolution: false` for all spots.

### TD-02: UpdateSolutionDto shape divergence (Phase 42)
Generated `UpdateSolutionDto` only has `description/metadata/title` while the backend may accept wider field set (`product_url/product_name/brand/price/currency/image_url`). Backend spec may be incomplete.
