# Requirements: decoded-monorepo v10.0 Tech Debt Resolution

**Defined:** 2026-03-26
**Core Value:** 프로덕션 트래픽을 안전하게 처리할 수 있는 안정성 — 메모리 누수 없음, 보안 취약점 없음, 프로덕션 오류 가시성 확보, 핵심 경로 테스트 커버리지

## v10.0 Requirements

### Memory & Stability (MEM)

- [ ] **MEM-01**: `URL.createObjectURL` 누수 수정 — `requestStore.ts`의 프리뷰 URL 생성을 컴포넌트의 `useEffect` cleanup으로 이동하여 언마운트 시 `revokeObjectURL` 보장 + `beforeunload` 안전망 추가
- [ ] **MEM-02**: GSAP context cleanup 수정 — `ImageDetailModal.tsx`에서 `useEffect` 내 ad-hoc `gsap.context()` 대신 `@gsap/react`의 `useGSAP()` 훅으로 교체하여 언마운트 시 `revert()` 자동 보장
- [ ] **MEM-03**: requestStore 상태 전환 가드 — `setStep()` 호출에 Zod 검증 추가 (1→2→3→4 순서 강제), `startDetection()` 실행 전 step 1 완료 필수 조건 추가
- [ ] **MEM-04**: Zustand 불변성 수정 — `setDetectedSpots()`에서 입력 spots를 `structuredClone()`으로 딥 클론하고 타입에 `Readonly<DetectedSpot[]>` 추가
- [ ] **MEM-05**: 스크롤 포워딩 완성 — `ImageDetailModal.tsx`의 미완성 스크롤 포워딩 구현을 완성하여 데스크탑에서 이미지 영역 스크롤이 콘텐츠 스크롤로 전달되도록 처리

### Security & Safety (SEC)

- [ ] **SEC-01**: `POST /api/v1/posts/analyze` 레이트 리밋 — 인증되지 않은 analyze 엔드포인트에 슬라이딩 윈도우 레이트 리밋 미들웨어 추가 (`lru-cache` 기반, IP당 10req/min)
- [ ] **SEC-02**: 디버그 로깅 제거 — `ImageDetailModal.tsx`, `authStore.ts`, API 프록시 라우트의 `console.log`/`console.error`를 `process.env.NODE_ENV === 'development'` 가드로 래핑하거나 민감 정보(토큰, URL) 제거
- [ ] **SEC-03**: 환경 변수 스타트업 검증 — `packages/web/lib/config/env.ts` 생성: `API_BASE_URL`, `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` 누락 시 모듈 로드 시점에 `throw Error` 실행
- [ ] **SEC-04**: API 프록시 구조적 오류 처리 — 모든 `/app/api/v1/**` 라우트 핸들러에서 백엔드 에러 코드(401, 422, 429)를 그대로 클라이언트에 전달하는 공유 `proxyHandler` 유틸리티 생성

### Component Refactoring (REF)

- [ ] **REF-01**: ThiingsGrid 물리 엔진 추출 — 948줄 `ThiingsGrid.tsx`에서 속도/마찰/그리드 셀 물리 엔진을 순수 클래스 `lib/utils/thiingsGridPhysics.ts`로 분리 (React 의존성 없음)
- [ ] **REF-02**: ImageDetailModal 애니메이션 추출 — 637줄 `ImageDetailModal.tsx`에서 GSAP 타임라인 오케스트레이션을 `useDrawerAnimation(ref, isOpen)` 커스텀 훅으로, 드로어 상태 전환을 `useDrawerState()` 훅으로 분리 → 메인 파일 200줄 이하로 감소
- [ ] **REF-03**: DecodedLogo WebGL 폴백 — `DecodedLogo.tsx`에 `canvas.getContext('webgl')` 감지 추가, `<ErrorBoundary>` 래핑, WebGL 미지원 환경(인스타그램/카카오톡 인앱 브라우저)용 SVG/정적 이미지 폴백 제공
- [ ] **REF-04**: CI 번들 사이즈 모니터링 — `size-limit` 또는 `bundlesize` devDependency 추가, `.size-limit.json` 임계값 설정 (메인 번들 < 500KB gzip), CI 빌드 후 `bun run size` 단계 추가

### Observability (OBS)

- [ ] **OBS-01**: Sentry 프론트엔드 에러 트래킹 — `@sentry/nextjs` 설치, `instrumentation.ts` + `sentry.client.config.ts` 초기화, PII 스크러빙 (`beforeSend`에서 토큰/이메일 제거), `tracesSampleRate: 0.1`
- [ ] **OBS-02**: Sentry Rust API 에러 트래킹 — `packages/api-server/Cargo.toml`에 `sentry@0.34+` 크레이트 추가, `main.rs`에서 Axum 시작 전 초기화, `sentry-tower` 미들웨어로 HTTP 요청에 trace 컨텍스트 연결
- [ ] **OBS-03**: Web Vitals 리포팅 — `packages/web/app/reportWebVitals.ts` 구현, LCP/FID/CLS 수집 후 Sentry performance 또는 커스텀 엔드포인트로 전송 (LCP 목표 <2.5s, CLS 목표 <0.1)
- [ ] **OBS-04**: 구조적 프록시 에러 전파 검증 — SEC-04 `proxyHandler`가 백엔드 422/401/429 응답을 Sentry에 올바르게 노출하는지 통합 테스트로 확인

### Testing (TEST)

- [ ] **TEST-01**: Vitest 스토어 단위 테스트 — `requestStore` 상태 전환 가드 테스트 (잘못된 step 전환 거부 확인), `authStore` 세션 초기화 및 에러 복구 테스트 (`vi.mock('@supabase/supabase-js')` 활용)
- [ ] **TEST-02**: Vitest 유틸리티 단위 테스트 — `apiToStoreCoord`/`storeToApiCoord` 엣지 케이스 테스트 (0%, 100%, 소수점 정밀도), REF-01 추출 후 `thiingsGridPhysics` 물리 엔진 단위 테스트
- [ ] **TEST-03**: E2E 테스트: 인증 + 모달 — Playwright로 OAuth 리다이렉트 → 세션 쿠키 흐름 및 이미지 디테일 모달 열기/닫기/Escape 키 + 빠른 연속 동작 테스트
- [ ] **TEST-04**: E2E 테스트: 업로드 흐름 — Playwright로 이미지 업로드 → AI 분석 → 결과 표시 전체 흐름 테스트; MEM-01 수정 후 `URL.createObjectURL` 누수 코드 경로 커버
