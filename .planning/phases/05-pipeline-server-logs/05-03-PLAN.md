**useServerLogs.ts:**
Follow useAiCost.ts pattern. Use the same `adminFetch` helper pattern.

- `useServerLogs(params: { level?: string; search?: string; from?: string; to?: string; page?: number; pageSize?: number })`: Fetches from `/api/v1/admin/server-logs?level=X&search=Y&from=ISO&to=ISO&page=N&pageSize=M`. Query key: `["admin", "server-logs", "list", level, search, from, page]`. Returns `UseQueryResult<ServerLogListResponse>`.

- `useLogStream()`: Custom hook (NOT just a React Query wrapper) that manages streaming state:
  - Internal state: `entries: ServerLogEntry[]` (accumulated), `latestId: string | null`, `isStreaming: boolean`, `isPaused: boolean`
  - Uses `useRef` for interval ID and scroll container ref
  - `start()`: Begins polling `/api/v1/admin/server-logs/stream?since_id=X` every 2500ms. Appends new entries to accumulated list. Updates latestId.
  - `pause()`: Stops polling interval but preserves accumulated entries
  - `resume()`: Restarts polling from latestId
  - `clear()`: Empties accumulated entries and resets latestId
  - `toggle()`: Toggles between pause/resume
  - Cap accumulated entries at 500 max (trim oldest when exceeded)
  - Returns: `{ entries, isStreaming, isPaused, toggle, clear, start }`
  - Auto-start on mount, cleanup interval on unmount
  - Import types from `@/lib/api/admin/server-logs`
