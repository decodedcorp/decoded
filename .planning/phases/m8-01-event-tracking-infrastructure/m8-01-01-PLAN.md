Supabase migration generator 스킬 패턴에 따라 SQL 마이그레이션 파일을 생성한다.

테이블: `public.user_events`
- id: uuid PK (gen_random_uuid)
- user_id: uuid NOT NULL, FK → public.users(id) ON DELETE CASCADE
- event_type: text NOT NULL (post_click, post_view, spot_click, search_query, category_filter, dwell_time, scroll_depth)
- entity_id: text (nullable — scroll_depth 등 엔티티 없는 이벤트 존재)
- session_id: text NOT NULL
- page_path: text NOT NULL
- metadata: jsonb (nullable)
- created_at: timestamptz NOT NULL DEFAULT now()

인덱스:
- GIN index on metadata (m8-02 affinity scoring용)
- Composite index on (user_id, created_at DESC) (per-user time-range 쿼리)
- Index on event_type (타입별 필터링)

RLS:
- authenticated 유저가 자신의 이벤트만 INSERT 가능 (auth.uid() = user_id)
- authenticated 유저가 자신의 이벤트만 SELECT 가능
- service_role은 full access (m8-03 어드민 쿼리용)

pg_cron TTL (주석 포함):
- 30일 이상 된 레코드 매일 03:00 UTC에 삭제
- pg_cron 미활성화 시 수동 실행 안내 주석 추가

updated_at 트리거는 불필요 — 이벤트는 불변(immutable) 데이터

MCP supabase__apply_migration으로 적용할 수 있도록 파일명 규칙 준수.
