## SCR-CREA-03-edit-solution.md

**PRE-VERIFIED FACTS (from actual code):**
- Spot editing and solution input happen WITHIN both /request/upload and /request/detect pages (no separate route)
- SolutionInputForm fields: title* (string, required), originalUrl* (string/url, required), priceAmount (number, optional), priceCurrency (string, default "KRW")
- Validation: isValid = title.trim() && originalUrl.trim()
- Submit flow (from upload page): compressImage → spots.map to {position_left, position_top, category_id} → createPostWithFile(FormData) → redirect
- categoryCodeMap from useCategoryCodeMap hook maps category code → UUID
- createPostWithFile sends FormData with "image" (File) + "data" (JSON: image_url, spots, media_source)
- resetRequestFlow clears all state + revokes preview URLs on completion or cancel
- canProceed for step 3: mediaSource?.type && mediaSource?.title (but upload page bypasses this with hardcoded {type:"youtube",title:"User Upload"})

**Structure:**
1. **Header** — ID: SCR-CREA-03, Route: (within `/request/upload` and `/request/detect`), Status: implemented
   - Note: This is not a separate route — spot editing and solution input are embedded within CREA-01 and CREA-02 pages
2. **Purpose** — User creates/edits spots, enters product solution info per spot, and submits the completed post
3. **Component Map** — Table (components used across both pages):
   - SolutionInputForm — verified path — title, URL, price fields
   - DetectedItemCard — verified path — card with expand/collapse for solution form
   - SpotMarker — verified path — re-exports DS Hotspot (deprecated wrapper)
   - useSpotCardSync — verified path — marker ↔ card scroll sync

4. **Spot Creation Interaction** — Step-by-step:
   1. User taps on image → DetectionView.handleImageClick fires
   2. Click position converted to normalized coords: x = (clientX - rect.left) / rect.width, clamped 0-1
   3. requestStore.addSpot(x, y) creates DetectedSpot with auto-generated id, incremented index
   4. New spot auto-selected (selectedSpotId = newSpot.id)
   5. Spot appears as numbered Hotspot marker on image + card in list

5. **Spot Removal** — removeSpot(spotId) → filter + re-index remaining spots

6. **SolutionInputForm Fields** — Table:
   | Field | Type | Required | Validation | UI |
   - title: string — Yes — trim() non-empty — text input, placeholder "Nike Air Force 1"
   - originalUrl: string (URL) — Yes — trim() non-empty — url input, placeholder "https://..."
   - priceAmount: number — No — integer parse — number input, placeholder "129000"
   - priceCurrency: string — No (default "KRW") — N/A — read-only display badge

7. **Submit Flow** — Ordered steps (from upload page handleNext):
   1. Guard: canProceed = detectedSpots.length > 0 && !isSubmitting
   2. Guard: categoryCodeMap.size > 0 (categories loaded)
   3. compressImage(localImage.file) — toast.loading "이미지 준비 중..."
   4. Map spots to API payload: { position_left: "XX.X%", position_top: "XX.X%", category_id: UUID }
   5. createPostWithFile({ file, media_source: {type:"youtube",title:"User Upload"}, spots }) — toast.loading "포스트 생성 중..."
   6. Success: toast.success → resetRequestFlow → router.push(/posts/[id])
   7. Failure: toast.error with Korean message

   ⚠️ NOTE: media_source is hardcoded to {type:"youtube",title:"User Upload"} — the DetailsStep/SubmitStep components exist but are NOT rendered in the current upload page flow.

8. **Reset Behavior:**
   - Close button (X) → resetRequestFlow + router.push("/")
   - Post success → resetRequestFlow + router.push(/posts/[id])
   - resetRequestFlow: revokes all preview URLs + resets to initialState

9. **requestStore Fields (Spot/Solution)** — Table:
   | Field | Type | Read/Write | Notes |
   - detectedSpots[].solution (SpotSolutionData|undefined) — write: setSpotSolution, clearSpotSolution
   - selectedSpotId (string|null) — write: selectSpot (null deselects after save)
   - isSubmitting (boolean) — write: local useState in upload page (NOT from store)
