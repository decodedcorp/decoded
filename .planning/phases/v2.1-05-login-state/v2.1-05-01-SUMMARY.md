# Phase v2.1-05 Plan 01: OAuthButton, GuestButton & Divider Summary

**One-liner:** OAuth buttons with Kakao/Google/Apple branding, guest button with transparent styling, and centered "or" divider for login flow

## What Was Built

### OAuthButton Component
CVA-based OAuth sign-in buttons with provider-specific branding:

| Provider | Background | Text | Hover |
|----------|------------|------|-------|
| Kakao | #FEE500 | #191919 | #FDD800 |
| Google | #FFFFFF | #374151 | gray-50 |
| Apple | #000000 | #FFFFFF | gray-900 |

Features:
- Height 52px, cornerRadius 12px per decoded.pen specs
- Inline SVG icons with correct brand colors
- Loading state with Loader2 spinner
- Disabled state styling

```tsx
import { OAuthButton } from "@/lib/design-system";

<OAuthButton provider="kakao" onClick={handleKakao} />
<OAuthButton provider="google" onClick={handleGoogle} isLoading />
<OAuthButton provider="apple" onClick={handleApple} />
```

### GuestButton Component
Transparent continuation button for guest users:

- Background: transparent
- Border: 1px white/20
- Text: white/70, hover white
- Height 52px, cornerRadius 12px

```tsx
import { GuestButton } from "@/lib/design-system";

<GuestButton onClick={continueAsGuest} />
<GuestButton>Skip for now</GuestButton>
```

### Divider Component
Horizontal divider with centered text:

- Lines: white/20
- Text: white/40, 12px font
- Default text: "or"

```tsx
import { Divider } from "@/lib/design-system";

<Divider />
<Divider text="or continue with" />
```

- cn() utility for className merging
- Consistent prop naming (isLoading, disabled)
