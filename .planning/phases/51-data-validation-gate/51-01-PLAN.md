Run three SQL queries against the Supabase database using `execute_sql` MCP tool. Record all results for Task 2.

**CRITICAL schema fact:** `post_magazine_title` is NOT a database column. The DB column is `post_magazine_id` (UUID, nullable) on the `posts` table. The `post_magazine_title` seen in API responses is derived at query time by the Rust API via a batch join to `post_magazines.title`. Always query `post_magazine_id IS NOT NULL`, never `post_magazine_title`.

**Query 1 — Gate count:**
```sql
SELECT COUNT(*) AS editorial_post_count
FROM posts
WHERE post_magazine_id IS NOT NULL;
```
Record the integer result. This is the primary gate: 0 = FAIL, 1+ = PASS.

**Query 2 — Status distribution:**
```sql
SELECT
  pm.status,
  COUNT(*) AS count,
  COUNT(pm.layout_json) AS with_layout
FROM posts p
JOIN post_magazines pm ON pm.id = p.post_magazine_id
WHERE p.post_magazine_id IS NOT NULL
GROUP BY pm.status
ORDER BY count DESC;
```
Record the full result table. Important: even if count > 0, if all rows have `status = 'generating'` or `status = 'failed'`, note this as a partial pipeline issue (title exists but `layout_json` is null — Phase 53 detail migration needs `layout_json`).

**Query 3 — Sample rows (up to 5):**
```sql
SELECT
  p.id AS post_id,
  p.post_magazine_id,
  pm.title AS post_magazine_title,
  pm.status AS magazine_status,
  CASE WHEN p.ai_summary IS NOT NULL THEN 'yes' ELSE 'no' END AS has_ai_summary,
  CASE WHEN pm.layout_json IS NOT NULL THEN 'yes' ELSE 'no' END AS has_layout_json,
  (SELECT COUNT(*) FROM spots s WHERE s.post_id = p.id) AS spot_count
FROM posts p
JOIN post_magazines pm ON pm.id = p.post_magazine_id
WHERE p.post_magazine_id IS NOT NULL
ORDER BY p.created_at DESC
LIMIT 5;
```
Record full table. Note which fields are present vs null for each row. `ai_summary` is on the `posts` table (NOT `post_magazines`).

If any query returns an error, record the exact error message — it may indicate a schema mismatch that needs investigation.
