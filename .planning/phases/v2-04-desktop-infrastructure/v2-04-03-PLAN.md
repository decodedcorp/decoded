  <files>packages/web/lib/components/ConditionalNav.tsx</files>
  <action>
Refactor ConditionalNav to use the new header components:

1. **Remove Sidebar Import:**
   - Remove `import { Sidebar } from "./Sidebar";`
   - Sidebar is no longer used in the new layout

2. **Add Design-System Imports:**
   ```typescript
   import { DesktopHeader, MobileHeader } from "@/lib/design-system";
   ```

3. **Update ConditionalNav Component:**
   ```typescript
   export function ConditionalNav() {
     return (
       <>
         {/* Desktop Header - visible on md+ */}
         <DesktopHeader />
         {/* Mobile Header - visible on <md */}
         <MobileHeader />
         {/* Mobile Bottom Nav - preserved */}
         <MobileNavBar />
       </>
     );
   }
   ```

4. **Update MainContentWrapper:**
   Remove the left margin offset since we no longer have a sidebar:
   ```typescript
   export function MainContentWrapper({
     children,
   }: {
     children: React.ReactNode;
   }) {
     return (
       <main className="min-h-screen pt-14 md:pt-16 pb-14 md:pb-0">
         {children}
       </main>
     );
   }
   ```

   Explanation:
   - `pt-14 md:pt-16`: Top padding for header height (56px mobile, 64px desktop)
   - `pb-14 md:pb-0`: Bottom padding for MobileNavBar on mobile only
   - Remove `md:ml-14 lg:ml-[200px]` - no sidebar offset needed
   - Remove `transition-[margin] duration-200` - no longer animating margin

5. **Search/Filter Handling (Temporary):**
   For now, the search and filter functionality will be handled at a higher level.
   The DesktopHeader and MobileHeader accept onClick handlers for search/filter.

   If search panel state management is needed, add local state:
   ```typescript
   const [isSearchOpen, setIsSearchOpen] = useState(false);
   // Pass handlers to headers
   ```

   Or defer this to v2-07 (Search & Image Detail) phase.

Add DesktopFooter to the root layout:

1. **Add Import:**
   ```typescript

   ```

2. **Update Layout Structure:**
   The footer should appear after the main content wrapper but still inside AppProviders.

   ```typescript
   export default function RootLayout({
     children,
     modal,
   }: {
     children: React.ReactNode;
     modal: React.ReactNode;
   }) {
     return (
       <html lang="en" suppressHydrationWarning>
         <body
           className={`${playfairDisplay.variable} ${inter.variable} font-sans`}
           suppressHydrationWarning
         >
           <AppProviders>
             {/* Conditional Navigation (Headers + MobileNav) */}
             <ConditionalNav />

             {/* Main Content Area */}
             <MainContentWrapper>
               {children}
               {modal}
             </MainContentWrapper>

             {/* Footer - after main content */}
             <DesktopFooter className="mt-auto" />
           </AppProviders>
         </body>
       </html>
     );
   }
   ```

3. **Flex Layout Consideration:**
   To ensure footer stays at bottom on short pages, consider wrapping in a flex container.

   Alternative approach - update body/wrapper:
   ```typescript
   <body className={`... min-h-screen flex flex-col`}>
     <AppProviders>
       <ConditionalNav />
       <MainContentWrapper>
         {children}
         {modal}
       </MainContentWrapper>
       <DesktopFooter />
     </AppProviders>
   </body>
   ```

   Or wrap the inner content:
   ```typescript
   <AppProviders>
     <div className="flex flex-col min-h-screen">
       <ConditionalNav />
       <MainContentWrapper>
         {children}
         {modal}
       </MainContentWrapper>
       <DesktopFooter className="mt-auto" />
     </div>
   </AppProviders>
   ```

   Use the wrapper div approach to avoid conflicts with existing body styles.

Note: The footer should have `mt-auto` to push to bottom when content is short.
