Create spot-card.tsx with CVA variants for 3 display contexts:

Props interface:
```typescript
interface SpotCardProps {
  imageUrl?: string;
  brand: string;
  name: string;
  price: string;
  variant?: "default" | "active" | "compact";
  onClick?: () => void;
  className?: string;
}
```

CVA Variants:

1. **default** (grid context):
   - Card: w-full, aspect-square image container
   - Image: object-cover, rounded-lg
   - Text below image: brand (xs, muted), name (sm, truncate), price (sm, font-medium)
   - Hover: subtle shadow, scale-[1.02]

2. **active** (detail panel context):
   - Card: larger, more prominent
   - Image: larger aspect ratio (3/4 or 4/5)
   - Text: larger font sizes
   - Border highlight: border-primary or ring-2 ring-primary
   - Optional: subtle glow effect

3. **compact** (list context):
   - Horizontal layout: flex-row
   - Image: small square (48-64px)
   - Text beside image: brand, name (line-clamp-1), price
   - Minimal padding

Implementation notes:
- Use forwardRef pattern
- Next.js Image with sizes prop for optimization
- Image fallback: bg-muted with placeholder
- onClick for modal trigger (NOT Link - per CONTEXT.md decision)
- Transition animations between variants (Claude's discretion on duration/easing)

Include SpotCardSkeleton with variant support.
