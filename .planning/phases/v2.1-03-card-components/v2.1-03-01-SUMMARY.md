# Phase [v2.1-03] Plan [01]: ArtistCard & StatCard Summary

**One-liner:** ArtistCard for profile grids with avatar fallback and StatCard for dashboard metrics with K/M number abbreviation

## What Was Built

Implemented two specialized card components for the decoded.pen design system:

1. **ArtistCard** - Profile card with avatar, name, and post count
   - Avatar rendering with Next.js Image optimization (64px circle)
   - Graceful fallback: First letter of name in gradient circle when no image
   - Click-to-profile navigation via Next.js Link
   - Post count with locale-aware formatting
   - Follows CVA + forwardRef pattern for consistency

2. **StatCard** - Dashboard statistics card with trend indicators
   - Formatted numeric value using formatStatValue (1.2K, 1.2M)
   - Optional trend indicator with direction (up/down/neutral)
   - Trend arrows using Lucide icons (TrendingUp/TrendingDown)
   - Direction-based colors: green (up), red (down), gray (neutral)
   - Optional onClick for interactive metric cards
   - Responsive text sizing (mobile/desktop)

3. **formatStatValue Utility** - Number abbreviation helper
   - Uses Intl.NumberFormat with compact notation
   - Handles K/M abbreviations (1,234 → "1.2K", 1,234,567 → "1.2M")
   - Numbers < 1,000 display with toLocaleString
   - Locale-aware and browser-native

### Number Formatting

formatStatValue uses native browser APIs for consistent formatting:

```typescript
export function formatStatValue(value: number): string {
  if (value >= 1_000_000 || value >= 1_000) {
    return new Intl.NumberFormat("en", {
      notation: "compact",
      maximumFractionDigits: 1,
    }).format(value);
  }
  return value.toLocaleString();
}
```

**Output examples:**
- 999 → "999"
- 1,234 → "1.2K"
- 1,234,567 → "1.2M"
- 1,000 → "1K"

## Decisions Made

### 1. Number Abbreviation Format

**Decision:** Use Intl.NumberFormat with compact notation

**Rationale:**
- Native browser API, no external dependencies
- Locale-aware formatting (supports internationalization)
- Consistent with web standards
- Handles edge cases automatically

**Alternatives Considered:**
- Custom string manipulation: Less maintainable, harder to test
- External library (numeral.js): Unnecessary dependency

**Impact:** All stat displays show abbreviated numbers, improving readability for large values

---

### 2. Artist Card Avatar Fallback

**Decision:** Show first letter of name in gradient circle

**Rationale:**
- Provides visual consistency across artist grids
- Personalized (shows user's initial)
- Avoids broken image states or generic icons
- Gradient background adds visual interest

**Alternatives Considered:**
- Generic user icon: Less personalized
- Empty circle: Poor visual hierarchy
- Default avatar image: Extra HTTP request

**Impact:** All artist cards gracefully handle missing avatars with personalized fallback

---

### 3. Stat Card Trend Colors

**Decision:** Green for up, red for down, gray for neutral

**Rationale:**
- Universal color convention (financial/analytics dashboards)
- Immediate visual recognition
- Accessible (supplemented by arrow icons)
- Consistent with user expectations

**Alternatives Considered:**
- Brand colors: Would reduce clarity
- Blue/orange: Less intuitive for trends
- No color coding: Harder to scan quickly

**Impact:** Consistent trend visualization across all stat displays, users instantly recognize positive/negative metrics
