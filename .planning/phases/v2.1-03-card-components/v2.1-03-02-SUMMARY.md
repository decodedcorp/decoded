# Phase v2.1-03 Plan 02: SpotCard & ShopCarouselCard Summary

**One-liner:** Product spot cards with 3 variant states (grid/detail/list) and horizontal shopping carousel cards with scroll-snap behavior for external links.

## What Was Built

Created two specialized card components for product display and shopping:

### 1. SpotCard Component
- **3 variants via CVA pattern:**
  - `default`: Grid context - square aspect ratio, text below image
  - `active`: Detail panel - 4:5 aspect ratio, ring-2 ring-primary highlight
  - `compact`: List context - horizontal layout with 64px thumbnail
- **Features:**
  - onClick handler for modal trigger (not navigation)
  - Next.js Image with responsive sizes optimization
  - Image fallback with gradient background
  - Hover effects: subtle shadow and scale-[1.02]
  - SpotCardSkeleton with variant support

### 2. ShopCarouselCard Component
- **Fixed width for carousel alignment:**
  - 180px on mobile
  - 220px on desktop
- **Features:**
  - External link (target="_blank" with noopener noreferrer)
  - CSS scroll-snap: snap-center class for parent carousel
  - Square aspect ratio image with centered text
  - Hover: subtle scale and shadow transition
  - ShopCarouselCardSkeleton matching dimensions
  - JSDoc with scroll-snap usage guide

### 3. Design System Integration
- Both components exported from `@/lib/design-system` barrel
- Consistent with existing card component patterns
- TypeScript types exported for consumer safety

## Decisions Made

**1. SpotCard onClick for Modal Trigger**
- Product spotting uses onClick, not Link/href
- Opens modal with product details, not navigation
- Aligns with CONTEXT.md decision on modal-based interaction

**2. Carousel Without Page Dots**
- ShopCarouselCard relies on native scroll-snap
- No controlled paging component needed
- Simpler implementation per CONTEXT.md

**3. Variant-Based Layout System**
- Single SpotCard component for all contexts
- Variant prop controls layout transformation
- Reduces code duplication vs. separate components

## Technical Approach

**CVA Pattern Consistency:**
- SpotCard follows Tag/ActionButton CVA pattern
- Variants defined with cva() for type safety
- forwardRef pattern for ref forwarding

**Image Optimization:**
- Next.js Image with sizes prop
  - SpotCard default: `(max-width: 768px) 50vw, (max-width: 1024px) 33vw, 16vw`
  - SpotCard active: `(max-width: 768px) 80vw, (max-width: 1024px) 50vw, 33vw`
  - ShopCarouselCard: `220px` fixed
- Gradient fallback for missing images

**Scroll-Snap Integration:**
- ShopCarouselCard includes snap-center class
- Parent container documentation in JSDoc
- Required parent classes: overflow-x-auto, snap-x snap-mandatory
