# Phase v2.1-03: Card Components - Research

**Researched:** 2026-02-06
**Domain:** React design system card components with CVA variants
**Confidence:** HIGH

## Summary

This phase implements four specialized card components (ArtistCard, StatCard, SpotCard, ShopCarouselCard) building on the existing decoded.pen design system foundation. The research confirms that the existing design system patterns (CVA variants, Card composition, forwardRef, skeleton patterns) established in v2.1-01 and v2.1-02 provide the complete technical foundation needed.

The standard approach combines:
- CVA (class-variance-authority) for variant management with Tailwind CSS classes
- Next.js Image component for optimized avatar/product images with fallbacks
- CSS scroll-snap for carousel behavior (native browser APIs over JS libraries)
- Intl.NumberFormat for number abbreviation (1K, 1M) without external dependencies
- Conditional Link vs onClick button patterns for navigation
- useState for card active state management

**Primary recommendation:** Use established design system patterns (CVA + Card composition + forwardRef) with native browser APIs (CSS scroll-snap, Intl.NumberFormat) to avoid unnecessary dependencies.

### Pattern 1: CVA Card Variants with forwardRef
**What:** Define card variants with CVA, compose with base Card, forward refs for DOM access
**When to use:** All card components in this phase
**Example:**
```typescript
// Source: Existing pattern from lib/design-system/tag.tsx and product-card.tsx
import { cva, type VariantProps } from "class-variance-authority";
import { cn } from "@/lib/utils";
import { forwardRef } from "react";
import { Card, CardContent } from "./card";

const artistCardVariants = cva(
  "flex flex-col items-center gap-3", // Base styles
  {
    variants: {
      size: {
        sm: "min-w-[100px]",
        md: "min-w-[120px]",
        lg: "min-w-[160px]",
      },
    },
    defaultVariants: {
      size: "md",
    },
  }
);

export interface ArtistCardProps
  extends React.HTMLAttributes<HTMLDivElement>,
    VariantProps<typeof artistCardVariants> {
  artistId: string;
  avatarUrl?: string;
  name: string;
  postCount: number;
}

export const ArtistCard = forwardRef<HTMLDivElement, ArtistCardProps>(
  ({ className, size, artistId, avatarUrl, name, postCount, ...props }, ref) => {
    return (
      <Card
        ref={ref}
        variant="default"
        size="sm"
        interactive
        className={cn(artistCardVariants({ size }), className)}
        {...props}
      >
        <CardContent className="p-4">
          {/* Card content */}
        </CardContent>
      </Card>
    );
  }
);

ArtistCard.displayName = "ArtistCard";
```

### Pattern 3: Number Abbreviation Utility
**What:** Format large numbers as 1K, 1.2M using Intl.NumberFormat
**When to use:** StatCard value display
**Example:**
```typescript
// Source: https://lingo.dev/en/javascript-i18n/format-numbers-compact-notation
export function formatNumber(value: number): string {
  return new Intl.NumberFormat("en-US", {
    notation: "compact",
    compactDisplay: "short",
    maximumFractionDigits: 1,
  }).format(value);
}

// Usage:
formatNumber(1000);    // "1K"
formatNumber(1234);    // "1.2K"
formatNumber(1000000); // "1M"
formatNumber(3500000); // "3.5M"
```

### Anti-Patterns to Avoid
- **Heavy carousel libraries (Swiper, react-slick):** CSS scroll-snap provides native, performant scrolling without bundle bloat
- **Custom number formatting logic:** Intl.NumberFormat is built-in and handles edge cases (negative numbers, decimals)
- **Global state for card active state:** Use useState in parent component for simpler, more predictable state management
- **Manual image optimization:** Always use Next.js Image component with fill and sizes for automatic optimization
- **Card composition without forwardRef:** Always forward refs for accessibility and third-party library integration

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Number abbreviation (1K, 1M) | Custom regex/logic | Intl.NumberFormat with notation: "compact" | Handles edge cases, i18n support, zero bundle size |
| Carousel snap scrolling | Custom scroll position tracking | CSS scroll-snap-type: x mandatory | Native hardware acceleration, touch support, accessibility |
| Image optimization | Manual srcset/sizes | Next.js Image with fill + sizes prop | Automatic WebP, responsive images, lazy loading |
| Avatar fallback initials | Manual string.charAt() logic | Existing getInitials() pattern | Already established in profile-header-card.tsx |
| Card hover transitions | Custom JavaScript animations | Tailwind transition-shadow duration-300 | CSS-based, performant, consistent with design system |
| Stat trend arrows | Custom SVG icons | Lucide React TrendingUp/TrendingDown | Consistent with existing icon usage, tree-shakeable |

**Key insight:** The design system already provides all necessary patterns (CVA variants, Card composition, Image optimization, number formatting via built-in APIs). No custom solutions needed.

## Common Pitfalls

### Pitfall 1: CSS Scroll-Snap Sensitivity on Mobile
**What goes wrong:** scroll-snap-type: x mandatory can feel too aggressive on mobile, snapping when user tries to scroll past
**Why it happens:** Default mandatory snapping captures all scroll events
**How to avoid:** For mobile-first designs, consider proximity snapping or tune with scroll-padding
**Warning signs:** User reports of "sticky" scrolling or difficulty browsing multiple cards

```css
/* If mandatory is too aggressive: */
.carousel {
  scroll-snap-type: x proximity; /* Less aggressive on mobile */
}

/* Or add scroll-padding for breathing room: */
.carousel {
  scroll-padding: 0 16px; /* 16px padding on sides */
}
```

### Pitfall 2: Next.js Image Without Sizes Prop
**What goes wrong:** Browser downloads unnecessarily large images for small avatars
**Why it happens:** Without sizes, Next.js defaults to 100vw, generating large srcset
**How to avoid:** Always specify sizes prop matching actual rendered size
**Warning signs:** Network tab shows 1200px images loading for 80px avatars

```typescript
// Bad:
<Image src={avatar} fill alt="..." />

// Good:
<Image src={avatar} fill sizes="80px" alt="..." />

// For responsive cards:
<Image
  src={product}
  fill
  sizes="(max-width: 768px) 50vw, (max-width: 1024px) 33vw, 25vw"
  alt="..."
/>
```

### Pitfall 3: Intl.NumberFormat Without maximumFractionDigits
**What goes wrong:** Numbers like 1234 show as "1.234K" instead of "1.2K"
**Why it happens:** Default precision shows all significant digits
**How to avoid:** Set maximumFractionDigits: 1 for cleaner display
**Warning signs:** Stat cards show "1.234K", "12.34K" instead of "1.2K", "12.3K"

```typescript
// Bad:
new Intl.NumberFormat("en-US", { notation: "compact" }).format(1234);
// Output: "1.234K"

// Good:
new Intl.NumberFormat("en-US", {
  notation: "compact",
  compactDisplay: "short",
  maximumFractionDigits: 1
}).format(1234);
// Output: "1.2K"
```

### Pitfall 4: SpotCard Variant State in Card Component
**What goes wrong:** Multiple SpotCards can be active simultaneously, or state doesn't sync with Hotspot clicks
**Why it happens:** Each card manages its own active state with useState
**How to avoid:** Lift state to parent component, pass variant as prop
**Warning signs:** Multiple cards show active ring, Hotspot clicks don't update card state

```typescript
// Bad - state in card:
const SpotCard = ({ item }) => {
  const [isActive, setIsActive] = useState(false); // ❌ Local state
  return <Card variant={isActive ? "active" : "default"} />;
};

// Good - state in parent:
const SpotGrid = () => {
  const [activeId, setActiveId] = useState<string | null>(null); // ✅ Lifted state
  return spots.map(spot => (
    <SpotCard
      variant={activeId === spot.id ? "active" : "default"}
      onClick={() => setActiveId(spot.id)}
    />
  ));
};
```

### Pitfall 5: Card Hover Shadow Without transition-shadow
**What goes wrong:** Hover effect appears instantly without smooth animation
**Why it happens:** Only shadow utility is applied, no transition property
**How to avoid:** Always pair hover:shadow-* with transition-shadow duration-300
**Warning signs:** Card shadows "pop" in/out instead of smooth fade

```typescript
// Bad:
<Card className="shadow-sm hover:shadow-lg">

// Good:
<Card className="shadow-sm hover:shadow-lg transition-shadow duration-300">

// Or set in CVA base styles:
const cardVariants = cva("transition-shadow duration-300", {
  variants: {
    interactive: {
      true: "hover:shadow-lg",
    },
  },
});
```

### StatCard with Number Formatting
```typescript
// Source: Intl.NumberFormat pattern + Card composition
"use client";

import { forwardRef } from "react";
import { TrendingUp, TrendingDown } from "lucide-react";
import { Card, CardContent } from "./card";
import { cn } from "@/lib/utils";

function formatStatValue(value: number): string {
  return new Intl.NumberFormat("en-US", {
    notation: "compact",
    compactDisplay: "short",
    maximumFractionDigits: 1,
  }).format(value);
}

export interface StatCardProps {
  label: string;
  value: number;
  trend?: {
    direction: "up" | "down";
    value: string;
  };
  className?: string;
}

export const StatCard = forwardRef<HTMLDivElement, StatCardProps>(
  ({ label, value, trend, className }, ref) => {
    const TrendIcon = trend?.direction === "up" ? TrendingUp : TrendingDown;
    const trendColor = trend?.direction === "up" ? "text-green-600" : "text-red-600";

    return (
      <Card ref={ref} variant="default" size="md" className={className}>
        <CardContent className="p-4 space-y-1">
          <p className="text-xs text-muted-foreground">{label}</p>
          <div className="flex items-baseline gap-2">
            <p className="text-2xl font-bold text-foreground">
              {formatStatValue(value)}
            </p>
            {trend && (
              <div className={cn("flex items-center gap-1 text-xs", trendColor)}>
                <TrendIcon className="w-3 h-3" />
                <span>{trend.value}</span>
              </div>
            )}
          </div>
        </CardContent>
      </Card>
    );
  }
);

StatCard.displayName = "StatCard";
```

### SpotCard with CVA Variants
```typescript
// Source: CVA pattern from tag.tsx + established Card composition
"use client";

import { forwardRef } from "react";
import Image from "next/image";
import { cva, type VariantProps } from "class-variance-authority";
import { Card, CardContent } from "./card";
import { cn } from "@/lib/utils";

export const spotCardVariants = cva(
  "transition-all duration-300", // Base transition
  {
    variants: {
      variant: {
        default: "cursor-pointer",
        active: "ring-2 ring-primary ring-offset-2",
        compact: "cursor-pointer",
      },
    },
    defaultVariants: {
      variant: "default",
    },
  }
);

export interface SpotCardProps
  extends React.HTMLAttributes<HTMLDivElement>,
    VariantProps<typeof spotCardVariants> {
  imageUrl: string;
  brand: string;
  name: string;
  price: number;
  onOpenDetail?: () => void;
}

export const SpotCard = forwardRef<HTMLDivElement, SpotCardProps>(
  ({ variant, imageUrl, brand, name, price, onOpenDetail, className, onClick, ...props }, ref) => {
    const handleClick = (e: React.MouseEvent<HTMLDivElement>) => {
      onClick?.(e);
      onOpenDetail?.();
    };

    const isCompact = variant === "compact";

    return (
      <Card
        ref={ref}
        variant="default"
        size="sm"
        interactive
        className={cn(spotCardVariants({ variant }), className)}
        onClick={handleClick}
        {...props}
      >
        <CardContent className={cn("p-3", isCompact && "flex gap-3")}>
          {/* Image */}
          <div className={cn(
            "relative bg-muted rounded overflow-hidden",
            isCompact ? "w-16 h-16 flex-shrink-0" : "aspect-square mb-2"
          )}>
            <Image src={imageUrl} alt={name} fill sizes={isCompact ? "64px" : "200px"} className="object-cover" />
          </div>

          {/* Info */}
          <div className={cn("space-y-0.5", isCompact && "flex-1 min-w-0")}>
            <p className="text-xs text-muted-foreground line-clamp-1">{brand}</p>
            <h4 className="text-sm font-medium text-foreground line-clamp-2">{name}</h4>
            <p className="text-sm font-bold text-foreground">₩{price.toLocaleString()}</p>
          </div>
        </CardContent>
      </Card>
    );
  }
);

SpotCard.displayName = "SpotCard";
```

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| JS carousel libraries (Swiper, react-slick) | CSS scroll-snap-type | 2019+ (CSS Scroll Snap spec) | Zero JS bundle, hardware acceleration, native touch support |
| Custom number formatting functions | Intl.NumberFormat with compact notation | 2020+ (ES2020 support) | Built-in i18n, zero bundle size, edge case handling |
| Manual srcset/picture tags | Next.js Image component | Next.js 10+ (2020) | Automatic optimization, WebP, responsive images |
| Component-level state for active cards | Parent-controlled variant props | React 16.8+ (Hooks) | Single source of truth, easier debugging, predictable state flow |
| CSS-in-JS hover animations | Tailwind transition utilities | Tailwind 3.0+ (2021) | Better performance, smaller bundle, design system consistency |

**Deprecated/outdated:**
- **react-number-format, numeral.js:** Intl.NumberFormat is now universally supported and built-in
- **react-snap-carousel (for basic use cases):** CSS scroll-snap provides equivalent functionality with zero dependencies
- **Custom hover shadow JavaScript:** Tailwind's transition-shadow + hover: utilities are sufficient and more performant
