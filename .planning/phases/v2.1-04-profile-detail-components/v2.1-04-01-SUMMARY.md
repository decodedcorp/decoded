## Implementation Notes

### Badge Silhouette Effect
Used CSS filters for locked state instead of separate images:
```tsx
className={cn(
  isLocked && "grayscale brightness-50"
)}
```

Overlay with HelpCircle icon provides "mystery" feel:
```tsx
{isLocked && (
  <div className="absolute inset-0 flex items-center justify-center bg-black/30 rounded-full">
    <HelpCircle className="w-8 h-8 text-white" strokeWidth={2.5} />
  </div>
)}
```

### Podium Medal System
Trophy icon (Lucide) with dynamic colors based on rank:
```tsx
const getMedalStyles = (rank: number) => {
  switch (rank) {
    case 1: return { bg: "bg-yellow-500/10", text: "text-yellow-500", icon: Trophy };
    case 2: return { bg: "bg-gray-400/10", text: "text-gray-400", icon: Trophy };
    case 3: return { bg: "bg-orange-600/10", text: "text-orange-600", icon: Trophy };
    default: return null;
  }
};
```

### Current User Highlight
Combines with podium styling (additive):
```tsx
className={cn(
  isPodium && medalStyles?.bg,
  isCurrentUser && "border-l-4 border-primary bg-primary/5",
)}
```

### Rank Change Indicator
Currently inline in LeaderItem (will be extracted to standalone RankingItem in Plan 02):
```tsx
{rankChange && rankChangeStyles && (
  <div className={cn("flex items-center gap-0.5", rankChangeStyles.color)}>
    <rankChangeStyles.icon className="w-3 h-3 md:w-4 md:h-4" />
    <span>{rankChange.value}</span>
  </div>
)}
```

## Decisions Made

1. **Badge locked state:** CSS filter (grayscale + brightness-50) + "?" overlay instead of separate locked images
   - **Why:** Reduces asset management, works with any badge icon
   - **Impact:** Consistent locked appearance across all badges

2. **Podium medal colors:** yellow-500 (gold), gray-400 (silver), orange-600 (bronze)
   - **Why:** High contrast, clear visual hierarchy, universally recognizable
   - **Impact:** Podium positions are immediately obvious

3. **Rank change inline (not extracted yet):** LeaderItem includes rank change indicator directly
   - **Why:** Simplifies Plan 01 scope, reduces component count initially
   - **Impact:** Will need to extract to RankingItem in Plan 02 for standalone use

4. **Current user highlight combines with podium:** Additive styling allows user to be podium and highlighted
   - **Why:** User should always be highlighted regardless of rank
   - **Impact:** Podium background + left border + subtle bg tint all work together
