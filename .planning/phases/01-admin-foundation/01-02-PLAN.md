**app/admin/layout.tsx** - Server component (admin-specific layout):

This layout replaces the root layout's ConditionalNav/ConditionalFooter for `/admin` routes. Since Next.js App Router nests layouts, we need to handle this properly:

1. This is a **server component** that performs a double-check of admin status:
   - Call `createSupabaseServerClient()` to get server Supabase client
   - Get session via `supabase.auth.getUser()`
   - If no user: `redirect("/")`
   - Call `checkIsAdmin(supabase, user.id)` from `lib/supabase/admin.ts`
   - If not admin: `redirect("/")`
   - If admin: render `AdminLayoutClient` with children

2. Pass admin user name to `AdminLayoutClient` for sidebar display

3. Important: The root `layout.tsx` renders `ConditionalNav`, `MainContentWrapper`, `ConditionalFooter`. We need to update `ConditionalNav` and `ConditionalFooter` to hide on `/admin` routes, and `MainContentWrapper` should not add padding for admin routes. Update the `ConditionalNav`, `ConditionalFooter`, and `MainContentWrapper` components:
   - Add `/admin` to their path exclusion checks (similar to how `/explore` and `/feed` are handled for footer)
   - ConditionalNav: Return null when pathname starts with `/admin`
   - ConditionalFooter: Add `/admin` to `HIDDEN_FOOTER_PATHS`
   - MainContentWrapper: Remove header padding when pathname starts with `/admin`
