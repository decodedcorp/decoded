       - Change `"packageManager": "yarn@4.9.2"` to `"packageManager": "bun@1.3.10"`
       - Change `"dev"` script from `"yarn workspace @decoded/web dev"` to `"bun run --filter @decoded/web dev"`
       - Change `"build"` script from `"yarn workspace @decoded/web build"` to `"bun run --filter @decoded/web build"`
       - Change `"start"` script from `"yarn workspace @decoded/web start"` to `"bun run --filter @decoded/web start"`
       - Change `"lint"` script from `"yarn workspace @decoded/web lint"` to `"bun run --filter @decoded/web lint"`

       - Change `"format:check"` script from `"yarn workspace @decoded/web format:check"` to `"bun run --filter @decoded/web format:check"`
       - Keep `"dependencies"` unchanged: `"class-variance-authority": "^0.7.1"`

    4. Create `bunfig.toml` at project root with this exact content:
       ```toml
       [install]
       # Use hoisted strategy for Next.js + Expo compatibility
       # Matches existing Yarn 4 nodeLinker: node-modules behavior
       strategy = "hoisted"

       # Force full hoisting for Expo Metro bundler compatibility
       # Metro cannot resolve packages under node_modules/.bun/ symlinks
       publicHoistPattern = ["*"]
       ```

       NOTE on peer-dependency extension: The .yarnrc.yml has a packageExtensions entry for @tailwindcss/postcss.
       Do NOT add `[install.peer-dependencies]` to bunfig.toml -- this is not a documented bun config section.
       Instead, bun's native resolver handles peer deps automatically. If the build fails with a peer dep warning
       for tailwindcss in Task 2 verification, add tailwindcss as a direct devDependency in packages/web/package.json
       as the fallback fix.

  </action>
