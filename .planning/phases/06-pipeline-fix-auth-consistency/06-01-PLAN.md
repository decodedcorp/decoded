```typescript
if (process.env.NODE_ENV !== "development") {
```

With the standard auth pattern (matching all other admin API routes):

```typescript
if (!user) {

  return NextResponse.json({ error: "Forbidden" }, { status: 403 });
}
```

This simply removes the `if (process.env.NODE_ENV !== "development")` wrapper. The middleware already handles dev-mode bypass for page-level access — API routes should always enforce auth consistently.

Note: The middleware (`packages/web/middleware.ts`) and admin layout (`app/admin/layout.tsx`) ALSO have dev bypasses. Those are intentional for page-level access and NOT part of this fix. Only the API routes need consistency.
