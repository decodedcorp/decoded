  <action>
Create a minimal GitHub Actions workflow for running visual QA tests on PRs:

```yaml
name: Visual QA

on:
  pull_request:
    branches: [main]
    paths:
      - 'packages/web/**'
      - '.github/workflows/visual-qa.yml'
  workflow_dispatch: # Allow manual trigger

jobs:
  visual-qa:
    name: Visual QA Screenshots
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'yarn'

      - name: Install dependencies
        run: yarn install --frozen-lockfile

      - name: Install Playwright browsers
        run: yarn playwright install chromium --with-deps
        working-directory: packages/web

      - name: Run visual QA tests
        run: yarn playwright test tests/visual-qa.spec.ts --project=chromium
        working-directory: packages/web
        env:
          CI: true

      - name: Upload screenshots
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: visual-qa-screenshots
          path: docs/qa-screenshots/
          retention-days: 30

      - name: Upload test results
        uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-report
          path: packages/web/test-results/
          retention-days: 7
```

Key design decisions:
- Only runs on PRs to main that touch web package (avoid unnecessary runs)
- Uses workflow_dispatch for manual triggering during debugging
- Installs only chromium browser (matches local config)
- Uploads screenshots as artifacts even on success (for review)
- Uploads test results only on failure (debugging)
- 15 minute timeout (generous for initial setup)
- Uses frozen lockfile for reproducibility

Note: This workflow assumes the dev server can start and serve pages.
If the project requires environment variables for the server, those need to be added as GitHub secrets.
