2. Restructure `packages/api-server/src/main.rs`:

   a. Replace `#[tokio::main] async fn main()` with a synchronous `fn main()` that:
      - Calls `sentry::init()` FIRST (before any async runtime)
      - Stores the returned `_guard` in main scope (must outlive async runtime)
      - Builds tokio runtime manually: `tokio::runtime::Builder::new_multi_thread().enable_all().build()?.block_on(async_main())`

   b. Move all existing async logic from main() into `async fn async_main() -> Result<(), Box<dyn std::error::Error>>`:
      - Config loading, tracing init, DB connection, migrations, router, server start
      - This is a pure move — no logic changes

   c. Sentry init configuration:
      ```rust
      let _guard = sentry::init((
          std::env::var("SENTRY_DSN").unwrap_or_default(),
          sentry::ClientOptions {
              release: sentry::release_name!(),
              environment: Some(
                  std::env::var("APP_ENV")
                      .unwrap_or_else(|_| "development".into())
                      .into(),
              ),
              traces_sample_rate: if cfg!(debug_assertions) { 1.0 } else { 0.1 },
              ..Default::default()
          },
      ));
      ```
      When SENTRY_DSN is empty string, sentry::init creates a disabled client — this is correct graceful degradation.

   d. Add sentry-tracing layer to the tracing_subscriber setup in async_main():
      After the existing `env_filter` and `fmt::layer()`, add the sentry_tracing layer:
      ```rust
      use sentry_tracing::EventFilter;

      // In both the JSON and text format branches:
      tracing_subscriber::registry()
          .with(env_filter)
          .with(fmt_layer)  // existing format layer
          .with(sentry_tracing::layer().event_filter(|md| {
              match *md.level() {
                  tracing::Level::ERROR => EventFilter::Exception,
                  tracing::Level::WARN => EventFilter::Breadcrumb,
                  _ => EventFilter::Ignore,
              }
          }))
          .init();
      ```
      NOTE: `sentry_tracing` is included via the `tracing` feature in the sentry crate — no separate crate needed.

   e. Add sentry-tower middleware layers to the Axum Router. Insert BEFORE the existing request_logger and CORS layers (outermost = first to execute):
      ```rust
      let app = axum::Router::new()
          .route("/health", axum::routing::get(handlers::health_check))
          .nest("/api/v1", api_router)
          .merge(SwaggerUi::new(...))
          .layer(sentry_tower::NewSentryLayer::<axum::extract::Request>::new_from_top())
          .layer(sentry_tower::SentryHttpLayer::with_transaction())
          .layer(axum::middleware::from_fn(middleware::request_logger_middleware))
          .layer(middleware::create_cors_layer(...))
          .with_state(state);
      ```
      Note: `sentry_tower` is re-exported from the sentry crate when `tower-axum-matched-path` feature is enabled. Use `sentry_tower::` prefix.

   CRITICAL constraints:
   - Do NOT use unwrap() on the SENTRY_DSN env var — use unwrap_or_default() for empty string fallback
   - The `_guard` variable MUST live in the synchronous main() function, NOT inside async_main()
   - Preserve ALL existing functionality — this is a restructure, not a rewrite

   At the top of `app.py`, add imports:
   ```python
   import os
   import sentry_sdk
   from sentry_sdk.integrations.fastapi import FastApiIntegration
   from sentry_sdk.integrations.starlette import StarletteIntegration
   from sentry_sdk.integrations.grpc import GRPCIntegration
   ```

   Add init function:
   ```python
   def _init_sentry() -> None:
       """Initialize Sentry SDK. Gracefully skips if SENTRY_DSN is not set."""
       dsn = os.environ.get("SENTRY_DSN", "")
       if not dsn:
           return
       sentry_sdk.init(
           dsn=dsn,
           environment=os.environ.get("APP_ENV", "development"),
           traces_sample_rate=0.1 if os.environ.get("APP_ENV") == "production" else 1.0,
           integrations=[
               StarletteIntegration(),
               FastApiIntegration(),
               GRPCIntegration(),
           ],
           send_default_pii=False,
       )
   ```

   Call `_init_sentry()` at the very beginning of `create_app()` in `bootstrap.py` (BEFORE FastAPI() constructor):
   ```python
   def create_app(application: Application) -> FastAPI:
       _init_sentry()  # Must run before FastAPI app creation
       # ... existing code ...
   ```

   OR if modifying bootstrap.py is cleaner, add the call there. The key requirement: `sentry_sdk.init()` MUST execute before `FastAPI(...)` is instantiated so the integration hooks are registered.

   IMPORTANT: The `_init_sentry()` function uses a guard pattern — if `SENTRY_DSN` env var is empty or missing, it returns early without initializing. This means local development without Sentry works unchanged.
