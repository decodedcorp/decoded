  <files>packages/web/lib/api/client.ts</files>
  <action>
Create a new `client.ts` file that provides a shared API client. Follow the pattern from `posts.ts` but extract it into a reusable module.

Requirements:
1. Export `getAuthToken()` function (extract from posts.ts)
2. Create `apiClient` function that wraps fetch with:
   - Automatic Authorization header injection when `requiresAuth: true`
   - Default Content-Type: application/json
   - Configurable base URL from `NEXT_PUBLIC_API_BASE_URL` or empty string
   - Standardized error handling that throws typed errors
3. Support both JSON body and FormData requests
4. Return typed response using generics

Interface should look like:
```typescript
interface ApiClientOptions {
  path: string;
  method?: 'GET' | 'POST' | 'PATCH' | 'DELETE';
  body?: unknown;
  requiresAuth?: boolean;
  headers?: Record<string, string>;
}

async function apiClient<T>(options: ApiClientOptions): Promise<T>
```

Error handling:
- Throw `ApiError` for non-2xx responses (use existing type from types.ts)
- Include response status and message in error
