```json

```

The `orval` binary is in devDependencies and bun resolves it from `node_modules/.bin/`. The `--config` flag points to `orval.config.ts` which is at the packages/web root (same directory as package.json).

Do NOT add a root-level package.json script — turbo runs tasks from the package directory by default. Root-level access uses `bun run --filter @decoded/web generate:api` or `turbo run generate`.

```json

  "$schema": "https://turbo.build/schema.json",

    "build": {
      "dependsOn": ["^build", "generate"],

}
```
