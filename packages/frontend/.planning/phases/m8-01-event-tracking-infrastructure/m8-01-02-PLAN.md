RESEARCH.md Pattern 1 + Pattern 2를 기반으로 Zustand behaviorStore를 생성한다.

타입 정의:
- EventType: 'post_click' | 'post_view' | 'spot_click' | 'search_query' | 'category_filter' | 'dwell_time' | 'scroll_depth'
- TrackEventPayload: { event_type: EventType; entity_id?: string; metadata?: Record<string, unknown> }
- EventQueueItem: TrackEventPayload + { timestamp: string; page_path: string; session_id: string }
- BehaviorState: { queue: EventQueueItem[]; track: (payload: TrackEventPayload) => void; flush: () => void }

getSessionId():
- sessionStorage 키: 'decoded_session_id'
- 없으면 crypto.randomUUID()로 생성 후 sessionStorage에 저장
- typeof window !== 'undefined' 가드 필수 (SSR 방지)

track(payload):
- EventQueueItem으로 확장 (timestamp, page_path, session_id 자동 주입)
- 큐에 추가
- 큐 길이 >= FLUSH_SIZE(20)이면 자동 flush()
- dev 환경에서 console.log('[track]', event_type, item)

flush():
- queue.length === 0이면 early return (Pitfall 1 방지)
- new Blob([JSON.stringify(queue)], { type: 'application/json' })
- navigator.sendBeacon('/api/v1/events', blob)
- set({ queue: [] })
- dev 환경에서 console.log('[flush] sending', queue.length, 'events')

initFlushTimer() 함수 (store 외부, 모듈 레벨):
- setInterval(30_000)으로 주기적 flush
- visibilitychange + pagehide 이벤트에 flush 바인딩
- cleanup 함수 반환
- typeof window !== 'undefined' 가드 (SSR 방지)
- export하여 layout.tsx 또는 providers.tsx에서 useEffect로 초기화 가능하도록

주의:
- React Query 사용 금지 (fire-and-forget에 부적합)
- queue를 localStorage에 저장 금지 (cross-tab 충돌)
- flush()를 컴포넌트 이벤트 핸들러에서 직접 호출 금지
- authStore를 behaviorStore에서 import 금지 (인증 확인은 useTrackEvent 훅에서)

3개의 트래킹 훅을 RESEARCH.md Pattern 3/4/5 기반으로 생성한다.

**useTrackEvent.ts** (Pattern 3):
- useBehaviorStore에서 track 셀렉터로 가져옴
- useAuthStore에서 user 확인
- user가 null이면 no-op 반환 (비로그인 유저 추적 안 함 — Claude's Discretion으로 logged-in only 결정)
- 반환: (payload: TrackEventPayload) => void
- 컴포넌트에서 직접 useBehaviorStore import 금지 — 반드시 이 훅을 통해 track

**useTrackDwellTime.ts** (Pattern 4):
- 파라미터: entityId: string
- ref를 반환 (HTMLElement에 attach)
- IntersectionObserver threshold: 0.5 (50% 가시 = "시청 중")
- isIntersecting → startRef.current = Date.now() + setTimeout(3000ms)
- 3초 후 track({ event_type: 'dwell_time', entity_id: entityId, metadata: { dwell_ms } })
- 3초 전에 뷰포트 이탈 → clearTimeout
- Pitfall 2 방지: 최초 1회 발화 후 observer.unobserve(el) 호출
- cleanup: observer.disconnect() + clearTimeout
- useTrackEvent() 사용하여 인증 가드 자동 적용

**useTrackScrollDepth.ts** (Pattern 5):
- 마일스톤: [25, 50, 75, 100]
- window scroll 이벤트 (passive: true)
- scrollY + innerHeight / scrollHeight * 100 → 퍼센트 계산
- firedRef = Set<number> — 각 마일스톤 1회만 발화 (Pitfall 3 방지)
- track({ event_type: 'scroll_depth', metadata: { depth_pct: milestone } })
- Claude's Discretion: per-milestone fire-once 방식 선택 (max-on-leave 아님)
- cleanup: removeEventListener
- useTrackEvent() 사용

모든 훅:
- 'use client' 없음 (hooks는 client component에서만 호출됨)
- useEffect 내에서 브라우저 API 접근 (SSR 안전)
