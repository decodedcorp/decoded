<decisions>
## Implementation Decisions

### NavBar 구조 & 아이템
- 현재 5개 아이템 구성 유지: Home, Search, Request, Feed, Profile
- Request(+) 버튼은 다른 아이템과 동일한 스타일 (특별 강조 없음)
- 아이콘 아래 라벨(텍스트) 표시 유지
- design-system 모듈로 이동: `lib/design-system/NavBar.tsx`, `NavItem.tsx`

### Active/Inactive 상태 표현
- Active 상태: primary 색상만 적용 (배경 하이라이트 없음)
- 탭 전환 시: 색상 페이드 애니메이션 (150ms transition)
- Inactive 상태: muted-foreground 색상 유지
- hover 상태: Claude 판단 (터치 디바이스에서는 hover 제거, 데스크톱에서만 적용)

### SectionHeader 스타일링
- motion 애니메이션 유지 (whileInView 페이드인 + 슬라이드업)
- VIEW MORE 링크: 텍스트만 (현재 스타일 유지)
- subtitle 옵션 유지
- design-system 모듈로 이동: `lib/design-system/SectionHeader.tsx`

### SearchTabs 탭 디자인
- 현재 탭 구성 유지: All, People, Media, Items (4개)
- 카운트 표시: 괄호 형식 "Label (count)"
- 슬라이딩 언더라인: spring 애니메이션 유지 (motion layoutId)
- design-system 모듈로 이동: `lib/design-system/Tabs.tsx` (범용화)

### Claude's Discretion
- hover 상태 처리: 디바이스별로 적절히 적용 (@media hover)
- NavItem 간격/패딩: decoded.pen 토큰에 맞게 조정
- Tabs 컴포넌트 범용화 수준: SearchTabs 외 다른 용도로도 사용 가능하게

</decisions>
