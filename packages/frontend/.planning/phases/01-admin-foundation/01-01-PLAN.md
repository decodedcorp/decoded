Create a Supabase client factory for Next.js middleware context. This is needed because the existing `server.ts` uses `cookies()` from `next/headers` which is not available in middleware. Middleware receives `NextRequest` and must use its cookies API instead.

Use `@supabase/auth-helpers-nextjs` `createMiddlewareClient` pattern:

```typescript
import { createMiddlewareClient } from "@supabase/auth-helpers-nextjs";
import type { NextRequest, NextResponse } from "next/server";
import type { Database } from "./types";
```

Export a function `createSupabaseMiddlewareClient(req: NextRequest, res: NextResponse)` that returns a typed Supabase client. This follows the same pattern as the existing `server.ts` but adapted for middleware context.

Also create a helper `checkIsAdmin(supabase: SupabaseClient<Database>, userId: string): Promise<boolean>` in a separate file `admin.ts` that:
1. Queries `users` table for the given `userId`
2. Returns `true` only if `is_admin === true`
3. Returns `false` on any error or missing record
4. Logs errors only in development mode

This is separated from middleware.ts so the admin check can be reused in layout-level server components for double-checking.

<task type="auto">
  <name>Task 2: Create Next.js middleware for admin route protection</name>
  <files>packages/web/middleware.ts</files>
  <action>
Create `packages/web/middleware.ts` (Next.js middleware at the package root, not inside `app/`).

The middleware must:

1. **Match only `/admin` routes** via the `config.matcher` export:
   ```typescript
   export const config = {
     matcher: ["/admin/:path*"],
   };
   ```

2. **Auth flow:**
   - Create Supabase middleware client using `createSupabaseMiddlewareClient(req, res)`
   - Get session via `supabase.auth.getSession()`
   - If no session: redirect to `/` (home) silently
   - If session exists: call `checkIsAdmin(supabase, session.user.id)`
   - If not admin: redirect to `/` silently
   - If admin: return `NextResponse.next()` to allow access

3. **Redirect behavior:**
   - Use `NextResponse.redirect(new URL("/", req.url))` for all denials
   - No toast, no error messages, no indication that admin panel exists
   - This matches the decision: "admin 패널 존재 자체를 비관리자에게 노출하지 않음"

4. **Session refresh:**
   - The middleware client automatically refreshes the session cookie
   - Return the response from `NextResponse.next()` to propagate cookie changes

Important: Do NOT use `@supabase/ssr` package (not in dependencies). Use `@supabase/auth-helpers-nextjs` which is already installed.
  </action>
