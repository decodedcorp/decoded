**usePipeline.ts:**
Follow useAiCost.ts pattern exactly. Use the same `adminFetch` helper pattern.
- `usePipelines(status?: string, page?: number, pageSize?: number)`: Fetches from `/api/v1/admin/pipeline?status=X&page=N&pageSize=M`. Query key: `["admin", "pipeline", "list", status, page]`. Returns `UseQueryResult<PipelineListResponse>`.
- `usePipelineDetail(id: string | null)`: Fetches from `/api/v1/admin/pipeline/{id}`. Query key: `["admin", "pipeline", "detail", id]`. Enabled only when id is truthy. Returns `UseQueryResult<PipelineExecution>`.
- Import types from `@/lib/api/admin/pipeline`

**PipelineStatusFilter.tsx:**
Follow audit StatusFilter.tsx pattern. Button group with options: All, Completed, Running, Failed. Highlighted active filter with same styling as audit status filter.
- Props: `{ value: string | undefined; onChange: (status: string | undefined) => void }`
- "All" sets value to undefined (no filter)

**PipelineTable.tsx:**
Table layout following AuditTable.tsx patterns.
- Props: `{ data: PipelineListItem[]; onSelectPipeline: (id: string) => void; expandedId: string | null }`
- Columns: Thumbnail (40x30 rounded), Post ID (truncated), Status (color badge), Duration (formatted), Triggered By, Started At (relative time)
- Status badges: completed = green bg + text, running = blue bg + pulse animation, failed = red bg + text
- Row click toggles expansion (sets expandedId)
- When a row is expanded, render `<PipelineDetail>` inline below the row (accordion style using a `<tr>` with `<td colSpan>`)
- For failed rows: show a "Retry" button in the actions column. On click: show sonner toast "Pipeline {id} retry initiated" and optimistically update status (mock only — no real API call)
- Export `PipelineTableSkeleton`: 5 shimmer rows matching column layout

**PipelineDetail.tsx:**
Inline accordion content showing step-by-step execution.
- Props: `{ pipelineId: string }` — fetches detail via `usePipelineDetail(pipelineId)`
- Renders 3 steps in a horizontal or vertical timeline layout:
  - Step icon: CheckCircle (completed/green), Loader (running/blue, spinning), XCircle (failed/red), Circle (pending/gray) — use lucide-react icons
  - Step name: "Upload", "Analyze", "Detect"
  - Duration: formatted as "X.Xs" or "—" if not available
  - Timestamps: started at, completed at (formatted)
  - For failed steps: error message in red text with a `bg-red-50 dark:bg-red-900/20` background box
  - For pending steps: grayed out text and icon
- Show a loading spinner while detail is fetching
- Use a vertical timeline with connecting lines between steps for visual clarity:
  - Each step is a row with an icon on the left, a vertical line connecting to the next step, and details on the right
